# engee-to-md

Инструмент для автоматической генерации технических описаний моделей [Engee](https://engee.com/) в формате Markdown с помощью ИИ.

## Что делает

Скрипт обходит папку с моделями (`.zip` / `.engee`), извлекает из каждой структуру блоков и подсистем, отправляет данные в языковую модель через OpenRouter и сохраняет готовое описание в `.md` файл.

Из каждой модели извлекается:
- имя модели и список файлов внутри архива
- блоки и соединения из `root.json` (основная схема)
- блоки и соединения из вложенных подсистем

Описание включает разделы: **Общее описание**, **Используемые блоки**, **Логическая цепочка**, **Структура модели**.

## Настройка

Передайте API-ключ одним из способов:

```bash
# Через переменную окружения
export OPENROUTER_API_KEY=sk-...

# Или через файл .env в корне проекта
OPENROUTER_API_KEY=sk-...
```

## Использование

1. Положите файлы моделей (`.zip` или `.engee`) в папку `models/`
2. Запустите скрипт:

```bash
python engee_to_md.py
```

3. Готовые `.md` файлы появятся в папке `mds_deepseek/`

Скрипт пропускает уже обработанные модели — повторный запуск продолжит с того места, где остановился.

## Структура проекта

```
.
├── engee_to_md.py      # основной скрипт
├── models/             # входные файлы моделей
├── mds_deepseek/       # выходные Markdown описания
└── .env                # API-ключ (опционально)
```

## Конфигурация

Основные параметры задаются в начале `engee_to_md.py`:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `INPUT_FOLDER` | `models` | Папка с моделями |
| `OUTPUT_FOLDER` | `./mds_deepseek` | Папка для результатов |
| `MODEL` | `deepseek/deepseek-chat` | Модель через OpenRouter |
